# 音频分片转录实现文档

## 一、概述

本文档描述了 AI 面试复盘系统中**大音频文件自动分片转录**功能的实现方案。

### 问题背景

小米 MiMo-v2.5-ASR 单次请求最大 **10 MB**（约 12 分钟音频），但面试录音通常 30+ 分钟。原始实现直接将整个音频文件发送给 ASR API，超过 10MB 时会返回 400 错误。

### 解决方案

在 `TranscriptionServiceImpl` 中加入智能分片逻辑：
- **小文件（≤ 6MB）**：直传 ASR，保持原有逻辑
- **大文件（> 6MB）**：自动预处理 → 静音检测切片 → 并发 ASR 调用 → 文本合并去重

### 技术栈

- **音频处理**：`javax.sound.sampled`（WAV）+ FFmpeg（MP3/其他格式）
- **静音检测**：纯 Java RMS 能量计算（零外部依赖）
- **并发控制**：`CompletableFuture` + `Semaphore`
- **文本去重**：最长公共子串（LCS）算法

## 四、组件详细说明

### 4.1 AudioChunkProperties — 配置属性

对应 `application.yml` 中的 `ai.chunk` 配置节：

```yaml
ai:
  chunk:
    max-direct-size: 6291456       # 6MB，小于该值直传
    min-segment-duration: 60       # 最小片段时长（秒）
    max-segment-duration: 600      # 最大片段时长（秒）
    min-silence-duration: 2.0      # 最小静音间隙（秒）
    overlap-duration: 1.5          # 重叠窗口（秒）
    concurrency: 2                 # 并发发送窗口
    max-retries: 3                 # 最大重试次数
    timeout-seconds: 30            # 单次请求超时
    ffmpeg-path: ${FFMPEG_PATH:ffmpeg}  # FFmpeg 路径
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|:-----|:-------|:-----|
| `max-direct-size` | 6MB | 小于该值的文件直传 ASR，不分片。考虑 base64 编码膨胀约 33%，6MB 原始文件编码后约 8MB，留 2MB 余量 |
| `min-segment-duration` | 60s | 防止因思考停顿产生过多小碎片。面试回答通常一段持续 1 分钟以上 |
| `max-segment-duration` | 600s | 防止一直无人声间隙导致超限。对应 10MB 的安全时长 |
| `min-silence-duration` | 2.0s | 连续静音达到此长度才被认为是"可切点"，自然的语句停顿 |
| `overlap-duration` | 1.5s | 覆盖常见单音节词和短暂停顿，避免切分点词语断裂 |
| `concurrency` | 2 | 同时允许的最大未完成 ASR 请求数 |
| `max-retries` | 3 | 单片段失败后的最大重试次数 |
| `timeout-seconds` | 30 | 单次 ASR 请求的超时时间 |
| `ffmpeg-path` | `ffmpeg` | FFmpeg 可执行文件路径。支持环境变量 `FFMPEG_PATH` 注入 |

### 4.2 FfmpegResolver — FFmpeg 路径智能探测器

**职责**：自动查找系统中可用的 FFmpeg 可执行文件，无需手动配置。

**三级搜索策略**：

```
第一级：配置路径（ai.chunk.ffmpeg-path / 环境变量 FFMPEG_PATH）
    ↓ 未配置或不可用
第二级：系统搜索（常见安装目录 + PATH 环境变量）
    ↓ 未找到
第三级：默认值 "ffmpeg"（依赖系统 PATH）
```

**Windows 搜索目录**：
- `C:\ffmpeg\bin\ffmpeg.exe`
- `C:\Program Files\ffmpeg\bin\ffmpeg.exe`
- `C:\tools\ffmpeg\bin\ffmpeg.exe`
- `C:\ProgramData\chocolatey\bin\ffmpeg.exe`
- `%USERPROFILE%\scoop\apps\ffmpeg\current\bin\ffmpeg.exe`
- `%PATH%` 中所有目录

**macOS 搜索目录**：
- `/usr/local/bin/ffmpeg`
- `/opt/homebrew/bin/ffmpeg`
- `$(brew --prefix ffmpeg)/bin/ffmpeg`

**Linux 搜索目录**：
- `/usr/bin/ffmpeg`
- `/usr/local/bin/ffmpeg`
- `/snap/bin/ffmpeg`
- `$PATH` 中所有目录

**验证机制**：对每个候选路径执行 `ffmpeg -version`，确认输出包含 "ffmpeg" 字样，防止误匹配其他程序。

### 4.3 AudioPreprocessor — 音频预处理器

**职责**：将任意格式音频转换为 16kHz 16bit 单声道 PCM WAV。

**实现策略**：
- **WAV 文件**：使用 `javax.sound.sampled`（Java 内置，零依赖）
- **MP3/其他格式**：通过 `FfmpegResolver` 获取 FFmpeg 路径，调用进行格式转换

**FFmpeg 调用命令**：
```bash
ffmpeg -i input.mp3 -ar 16000 -ac 1 -sample_fmt s16 -y output.wav
```

**环境要求**：
- WAV 文件：无额外要求
- MP3/M4A/OGG/FLAC 等：需要 FFmpeg（由 FfmpegResolver 自动探测路径）

**错误处理**：
- FFmpeg 未找到时，抛出明确的安装指引
- 转换失败时，记录 FFmpeg 输出日志便于排查

### 4.4 SilenceDetector — 静音检测器

**职责**：基于 PCM 帧的 RMS 能量检测音频中的静音区域。

**算法**：
1. 将 PCM 数据按 20ms 帧切分
2. 计算每帧的 RMS（均方根）能量
3. RMS < 500 的帧判定为静音
4. 连续静音帧 ≥ `minSilenceDuration` 的区域记录为静音区域

**输出示例**：
```
[
  SilenceRegion(startTime=45.2, endTime=47.8),  // 2.6秒静音
  SilenceRegion(startTime=120.0, endTime=123.5)  // 3.5秒静音
]
```

### 4.5 AudioSplitter — 智能切片器

**职责**：融合静音检测 + 最小/最大时长约束 + 重叠窗口，将长音频切分为多个片段。

**切分策略**：
1. 当缓冲时长 ≥ `minSegmentDuration` 且遇到静音区域时，在静音中点切分
2. 当缓冲时长 ≥ `maxSegmentDuration` 时，即使没有静音也强制切分
3. 每个片段附加上一片段尾部 `overlapDuration` 秒作为上下文重叠

**重叠窗口的作用**：
- 避免切分点处的词语被截断
- ASR 在重叠区会产生重复识别，由 TextMerger 去重

**输出**：`List<AudioChunk>`，每个包含 seq、WAV 字节、起止时间

### 4.6 ReliableSender — 可靠并发发送器

**职责**：将多个音频片段并发发送给 ASR 客户端，自动重试失败的片段。

**核心机制**：
- **序列号（seq）**：每个片段唯一且递增
- **已确认集合（acked）**：`ConcurrentHashMap` 保证幂等，防止重复
- **并发窗口**：`Semaphore` 控制同时进行的 ASR 请求数
- **超时重试**：5xx 错误或异常时重试，4xx 错误放弃
- **指数退避**：重试间隔 = 2秒 × 尝试次数

**流程**：
```
for each chunk:
    acquire semaphore
    if already acked: skip
    call ASR → success → ack & store result
    call ASR → 5xx → retry with backoff
    call ASR → 4xx → give up
    release semaphore
```

### 4.7 TextMerger — 文本合并去重器

**职责**：将多个片段的 ASR 转录文本按 seq 排序合并，去除重叠区重复。

**去重算法**：
1. 取前一片段尾部 N 个字符作为搜索窗口
2. 取当前片段开头 N 个字符作为搜索窗口
3. 用动态规划寻找最长公共子串（LCS）
4. 如果 LCS ≥ 3 字符，裁剪掉当前片段的重复前缀

**参数**：
- N = `overlapDuration × 4 + 5`（中文约 4 字/秒）

## 五、TranscriptionServiceImpl 改造

### 改造前
```java
public String transcribe(String audioFilePath, Long configId) {
    byte[] audioBytes = Files.readAllBytes(path);
    MultipartFile multipartFile = new SimpleMultipartFile(filename, audioBytes);
    return client.transcribe(multipartFile, "zh");
}
```

### 改造后
```java
public String transcribe(String audioFilePath, Long configId) {
    byte[] audioBytes = Files.readAllBytes(path);
    AsrClient client = aiClientFactory.getAsrClientByConfigId(configId);

    // 小文件直传
    if (audioBytes.length <= chunkProperties.getMaxDirectSize()) {
        return client.transcribe(new SimpleMultipartFile(filename, audioBytes), "zh");
    }

    // 大文件：分片处理
    byte[] wavData = audioPreprocessor.preprocess(audioBytes, filename);
    List<AudioChunk> chunks = audioSplitter.split(wavData);
    Map<Integer, String> results = reliableSender.sendAll(chunks, client);
    return textMerger.merge(results, audioSplitter.getOverlapDuration());
}
```

## 六、使用方式

### 6.1 基本使用

无需额外操作。当用户上传超过 6MB 的音频文件时，系统自动进入分片转录流程。

### 6.2 配置调整

在 `application.yml` 中调整 `ai.chunk` 配置节：

```yaml
ai:
  chunk:
    max-direct-size: 4194304       # 改为 4MB
    min-segment-duration: 30       # 改为 30 秒
    concurrency: 3                 # 增加并发数
    ffmpeg-path: D:/ffmpeg/bin/ffmpeg.exe  # 指定 FFmpeg 完整路径
```

### 6.3 FFmpeg 路径配置

处理 MP3/M4A/OGG/FLAC 等非 WAV 格式时需要 FFmpeg。系统采用**智能路径探测**，大多数情况下无需手动配置。

**自动探测（推荐）**：

系统启动时会自动按以下顺序搜索 FFmpeg：
1. 检查配置路径（`ai.chunk.ffmpeg-path` 或环境变量 `FFMPEG_PATH`）
2. 遍历常见安装目录（如 `C:\ffmpeg\bin\`、`/usr/local/bin/` 等）
3. 搜索系统 PATH 环境变量

只要 FFmpeg 已安装，系统会自动找到它。

**手动配置（特殊场景）**：

如果自动探测失败，可通过以下方式指定路径：

```bash
# 方式一：环境变量
export FFMPEG_PATH=/usr/local/bin/ffmpeg        # Linux/macOS
set FFMPEG_PATH=D:\ffmpeg\bin\ffmpeg.exe        # Windows CMD

# 方式二：application.yml
ai:
  chunk:
    ffmpeg-path: D:/ffmpeg/bin/ffmpeg.exe
```

**优先级**：配置路径 > 系统搜索 > 默认值 `ffmpeg`

### 6.4 环境要求

| 音频格式 | 环境要求 |
|:---------|:---------|
| WAV | 无额外要求 |
| MP3 | 需要 FFmpeg |
| M4A/OGG/FLAC/WebM | 需要 FFmpeg |

**安装 FFmpeg**：
- Windows: 从 https://ffmpeg.org/download.html 下载，解压后配置 `ffmpeg-path` 或添加到 PATH
- Linux: `sudo apt install ffmpeg` 或 `sudo yum install ffmpeg`
- macOS: `brew install ffmpeg`

## 八、异常处理

| 异常场景 | 处理方式 |
|:---------|:---------|
| 音频文件不存在 | 抛出 `BusinessException`，提示"音频文件不存在" |
| FFmpeg 未安装（非 WAV 格式） | 抛出 `BusinessException`，提示安装 FFmpeg |
| FFmpeg 转换失败 | 记录 FFmpeg 输出日志，抛出异常 |
| 单片段 ASR 调用失败 | 自动重试最多 3 次，指数退避 |
| 某片段最终失败 | 记录警告日志，跳过该片段，合并其余文本 |
| 所有片段失败 | 抛出 `BusinessException` |

## 九、性能预估

以 30 分钟面试录音为例（16kHz 16bit 单声道 WAV）：

| 指标 | 数值 |
|:-----|:-----|
| 原始文件大小 | ~57.6 MB |
| 切片数量 | 3-5 个片段 |
| 每片段大小 | ~10-15 MB WAV |
| ASR 调用次数 | 3-5 次（含重试可能 4-8 次） |
| 预计耗时 | 1-3 分钟（取决于网络和 ASR 响应速度） |
| 内存占用 | ~120 MB（WAV 数据 + PCM 缓冲） |
